# Uncle Bob Reference: Clean Code

**Source:** `book_Robert_C_Martin_2008_Clean_Code_A_Handbook_of_Agile_Software_Craftsmanship.pdf`
**Author:** Robert C. Martin (Uncle Bob)

## Core Concepts
- **The Boy Scout Rule:** Leave the code cleaner than you found it.
- **SRP (Single Responsibility Principle):** A module should have one, and only one, reason to change.
- **Intention-Revealing Names:** Names should tell you why it exists, what it does, and how it is used.
- **F.I.R.S.T Rubric for Tests:** Fast, Independent, Repeatable, Self-Validating, Timely.
- **Small Functions:** Functions should do one thing and do it well.
